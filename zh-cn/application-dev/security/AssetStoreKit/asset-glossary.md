## A

- ### Alias；别名

    关键资产的索引标识，与业务身份共同构成关键资产的唯一索引，业务需保证每条关键资产的别名唯一。

- ### Asset；关键资产

    长度不超过1KB的用户短敏感数据，如密码、令牌、银行卡号等。由业务写入关键资产存储服务，以密文形式存储在ASSET数据库中。每条关键资产包含明文、别名及多个可自定义的附属属性。

- ### Asset Owner；关键资产属主

    写入关键资产的业务，关键资产只允许被其属主访问，属主身份由ASSET从系统服务中获取，即使业务身份被仿冒也无法获取其他业务的数据。

- ### Asset Store Kit (ASSET)；关键资产存储服务

    OpenHarmony提供的短敏感数据安全存储及管理服务，包括新增、删除、更新、查询等操作，底层依赖通用密钥库系统在安全环境中完成关键资产的加/解密与访问控制。

- ### AuthToken；授权令牌

    统一用户认证框架签发的用户身份认证的通过证明，用于在查询需要用户认证的关键资产时与挑战值一同传入，以完成身份验证并授权关键资产的访问。

- ### Auth Type；用户认证类型

    访问关键资产所需的用户认证方式，支持NONE（无需认证）和ANY（任意一种认证方式通过即可访问），认证方式包括PIN码、人脸、指纹等。

- ### Auth Validity Period；认证有效期

    用户认证的有效时间窗口，最长可设置10分钟，在有效期内一次认证可授权多个关键资产的访问，无需重复认证。

## C

- ### Challenge；挑战值

    查询预处理时生成的随机值，用于防止用户认证过程中的重放攻击，在后续查询和后置处理中需传入此挑战值。

- ### Conflict Resolution；冲突处理策略

    新增关键资产时遇到别名相同冲突的处理方式，可选择OVERWRITE（覆盖原有资产）或THROW_ERROR（抛出异常由业务处理）。

## D

- ### Data Label；附属信息

    关键资产的自定义附加数据字段，分为有完整性保护的CRITICAL类（写入后不可更新）和无完整性保护的NORMAL类（可更新），ASSET为业务预留了12个此类属性，名称以DATA_LABEL开头。

- ### Developer ID；开发者标识

    标识开发者的唯一信息，与群组标识共同构成关键资产所属的群组信息，由系统自动获取，无需配置。

## G

- ### Group-based Access Control；群组访问控制

    允许同一开发者开发的多个应用配置为同一群组，群组内的应用可以互通访问彼此的关键资产数据，实现群组级别的数据共享。

- ### Group ID；群组标识

    标识关键资产所属群组的字段，由开发者在应用配置文件中配置，与开发者标识共同构成群组信息，用于实现群组内应用间关键资产的互通访问。

## I

- ### Integrity Protection；完整性保护

    对关键资产部分属性（名称以DATA_LABEL_CRITICAL开头的属性）提供的防篡改保护机制，受完整性保护的属性写入后不可更新，确保附属信息不被非法篡改。

## O

- ### Owner-based Access Control；属主访问控制

    所有关键资产默认受属主访问控制保护，只允许关键资产被其属主访问，属主身份参与关键资产加/解密的完整性保护。

## P

- ### Password-based Access Control；锁屏密码访问控制

    在用户设置了锁屏密码后才允许访问关键资产的访问控制策略，默认不开启，业务可根据需要设置。

- ### Persistent Asset；持久化关键资产

    在应用卸载时仍保留的关键资产，需申请ohos.permission.STORE_PERSISTENT_DATA权限，IS_PERSISTENT设为true的关键资产不允许设置为群组共享。

- ### PostQuery；查询后置处理

    查询需要用户认证的关键资产后的后置处理步骤，用于释放PreQuery占用的缓存资源，须与PreQuery成对调用，否则会导致缓存数量超限。

- ### PreQuery；查询预处理

    查询需要用户认证的关键资产前的预处理步骤，用于获取挑战值以供后续用户认证和查询使用，须与PostQuery成对调用以释放资源。

## R

- ### Return Type；查询返回类型

    关键资产查询返回的结果类型，ALL返回关键资产明文及属性，ATTRIBUTES仅返回属性不含明文。

## S

- ### Screen Lock-based Access Control；锁屏状态访问控制

    基于设备锁屏状态的访问控制策略，分为开机后可访问、首次解锁后可访问、解锁时可访问三个等级，安全性依次递增，默认为首次解锁后可访问。

- ### Secret；秘密明文

    关键资产中存储的用户敏感数据的秘密原文，如密码、令牌、银行卡号等，查询明文需解密操作时间较长，仅查询属性不需解密时间较短。

- ### Sync Type；同步类型

    关键资产支持的跨设备同步类型，包括NEVER（不允许同步）、THIS_DEVICE（仅本设备同步）、TRUSTED_DEVICE（可信设备间同步）、TRUSTED_ACCOUNT（可信账号间同步）。

## T

- ### Trusted Account；可信账号

    用于关键资产云同步场景中登录的可信账号，仅在该账号登录的设备间进行关键资产同步。

- ### Trusted Device；可信设备

    用于关键资产克隆场景中信任的设备，仅在新旧可信设备间进行关键资产同步。

## U

- ### User Authentication-based Access Control；用户认证访问控制

    关键资产在用户身份认证通过后被允许访问的访问控制策略，任意一种认证方式（指纹、人脸、PIN码）通过即可授权访问，可设置认证有效期实现一次认证授权多次访问。

## W

- ### Wrap Type；加密导入导出类型

    关键资产支持的加密导入导出类型，NEVER表示不允许加密导入导出，TRUSTED_ACCOUNT表示仅在可信账号登录的设备间进行加密导入导出。
