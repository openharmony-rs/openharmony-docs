使用NDK开发UI界面时，通过为组件添加事件监听和事件处理逻辑，可以让UI界面响应各种交互事件，包括基础输入事件、手势事件以及拖拽事件。

## 添加事件监听

NDK接口为组件提供了事件监听能力，开发者可以通过添加事件监听器并注册事件类型来接收和处理事件。

### 监听方式

NDK接口支持以下两种事件监听方式。

- **监听节点事件**：针对单个组件添加事件监听器，只监听该组件上发生的事件。使用[addNodeEventReceiver](../reference/apis-arkui/capi-arkui-nativemodule-arkui-nativenodeapi-1.md#addnodeeventreceiver)添加监听器，然后使用[registerNodeEvent](../reference/apis-arkui/capi-arkui-nativemodule-arkui-nativenodeapi-1.md#registernodeevent)声明需要监听的事件类型。
- **监听全局事件**：监听所有Native组件的事件触发。使用[registerNodeEventReceiver](../reference/apis-arkui/capi-arkui-nativemodule-arkui-nativenodeapi-1.md#registernodeeventreceiver)注册全局的事件处理函数，能够统一处理各个组件的事件。

除了组件交互事件外，NDK接口还支持监听组件布局完成事件、绘制送显事件以及系统深浅色变更事件等。

## 绑定交互事件

在添加事件监听的基础上，开发者可以绑定各种交互事件来响应用户的输入行为。交互事件主要分为基础输入事件、手势事件和拖拽事件三类。

### 基础输入事件

基础输入事件是用户通过不同外设（如触摸屏、鼠标、键盘等）直接生成的原始事件。根据事件特点总体上分为以下两类。

- **指向性事件**：包含事件发生位置信息的事件，以此次交互的第一个事件发生时所命中的组件来确定派发目标。包括：触摸事件、鼠标事件、轴事件。
- **非指向性事件**：没有具体位置信息的事件，以当前焦点组件为派发目标。包括：按键事件。

基础输入事件通常以一个标志开始的事件作为起始，以一个标志结束的事件作为结尾，中间持续上报移动或更新事件。对于指向性事件，系统会尽可能保证参与响应的组件可以得到完整的序列，但对于非指向性事件，由于焦点的变化，开始与结束可能被分发到不同的组件上。

基础输入事件通过响应链进行传递，事件从最内层组件优先处理，再逐层往父组件传递，任意一层组件可主动终止事件的继续传递。

### 手势事件

手势是一系列基础输入事件不断上报积累后，达成一定特点时所被识别成的交互结果，如点击：按下并在较短时间内抬起。

手势能够屏蔽不同基础输入事件的差异，提供更高级的交互能力。NDK接口支持单一手势和组合手势两种类型。单一手势包括点击、滑动、长按、捏合、旋转、快滑等。组合手势可将多个不同类型的手势组合在一起，通过对组内手势设置识别模式（顺序、并行、互斥）来控制手势之间的关系。

一个组件上可绑定多个手势，这些手势会在用户按下时，通过触摸测试被收集上来，由系统统一管理，所有手势都会持续接收到输入事件，直到有一个手势满足条件，在这之后，就只有这一个手势可以继续接收和处理输入事件。此外，NDK接口还支持自定义手势判定能力，可在手势识别器即将触发成功时动态决策是否让该识别器被系统认定为识别成功。

### 拖拽事件

拖拽事件是指用户将某个对象从源位置移动到目标位置的交互行为。NDK接口提供了以下两种拖拽方式。

**通用拖拽：** 通过组件的拖拽属性设置，使组件具备可拖拽能力。在拖拽的不同阶段会触发相应的组件事件（如预拖拽、起拖、进入、移动、离开、落入、结束等），支持自定义拖拽预览图和跟手图选项，并通过UDMF（统一数据管理框架）进行数据传递。

**DragAction主动发起拖拽：** 通过事件主动发起拖拽的能力。通过注册[NODE_ON_TOUCH_INTERCEPT](../reference/apis-arkui/capi-native-node-h.md#arkui_nodeeventtype)事件，在接收到触摸事件后创建DragAction对象，设置跟手图选项、跟手点和拖拽数据，然后主动发起拖拽。这种方式提供了更灵活的拖拽控制，可实现按需触发拖拽的场景。
