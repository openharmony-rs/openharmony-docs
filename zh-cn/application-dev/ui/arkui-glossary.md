## A

### Accessibility Virtual Node；无障碍虚拟子节点

当开发者使用自绘制组件进行自定义绘制时，这些组件内部的内容（如绘制的图形、文字）通常无法被无障碍服务直接感知。无障碍虚拟子节点是为自绘制组件补充的不可见无障碍节点，用于让辅助应用识别自绘区域内的可感知内容。

### Affine Property；仿射属性

组件的变换类属性（缩放、旋转、平移），基于组件坐标系和锚点进行仿射变换。组件平移后仿射属性的坐标系仍保持固定，与布局类属性（宽、高等）直接改变内容位置的方式不同。变更仿射属性不会触发测量布局，性能开销低于布局属性变更。

### Animatable Property；可动画属性

当属性值在可动画的闭包函数内发生变化时产生平滑的动画过渡而非直接跳变。区别于不可动画属性变化时瞬间生效无过渡。

### Axis Event；轴事件

由鼠标滚轮或触控板等带指针输入设备，在指针位于组件区域内时因滚动滚轮、双指沿水平、垂直轴滑动或双指捏合而触发的通用事件。“轴”指二维方向（水平X轴、垂直Y轴），事件携带水平轴值、垂直轴值和缩放比例，用于实现自定义滚动、缩放交互。

## C

### Concurrent Recognition；并发识别

组合手势的一种识别模式，组内各手势同时独立识别、互不影响，直到全部识别结束。用于让多个手势在同一时段并行响应（如同时缩放与旋转）。

### Crown Event；表冠事件

用户旋转穿戴设备的物理表冠时触发的通用事件，事件分发依赖应用焦点（仅获焦组件可接收），仅穿戴设备支持。携带旋转角速度、相对旋转角度、动作（开始、继续、结束）等信息，常用于驱动列表滚动、选择器调节。

## D

### dirty nodes；脏节点

状态变量变化后被标记为需要更新的关联节点，会在下一帧中更新并从脏节点列表清空；用于描述当前仍等待刷新的组件或节点状态。

### dirty state marking；标脏

组件状态或属性变化后，框架将对应组件或节点标记为需要更新的过程，用于在后续UI渲染中刷新标脏节点，并触发相关测量、布局和渲染；自定义属性或自定义绘制属性变化后，开发者需要主动通知框架重新测算或渲染。

### Drag Event；拖拽事件

用户在界面上拖动对象（如文件、控件）时触发的通用事件序列，包括拖拽开始、进入目标、移动、离开目标、释放（落入）、结束等阶段。支持自定义拖拽预览图、拖拽数据、落位动效、跨设备拖拽、悬停检测等。

### Drag Preview；拖拽预览图

拖拽过程中跟随手指或鼠标移动、用于表征被拖拽对象的预览图像，可由自定义构建器、拖拽信息对象或组件截图构成，支持配置缩放、阴影、圆角、材质等样式及数量角标。是拖拽交互中向用户传递拖拽内容的核心视觉载体。

## E

### Entity Node；实体节点

由后端创建的Native节点，是前端节点对象在底层的实际载体。前端节点通过引用关系与实体节点绑定，当调用dispose解除绑定后，前端节点不再对应任何实体节点，再次调用接口可能导致crash或返回默认值。

### Event Monopolize；事件独占

组件在本次交互中率先响应事件后，独占当前窗口内本次交互全部事件响应权的能力。启用后窗口内其他组件（含父子组件）的事件均不响应，用于避免手势或触摸事件在重叠组件间产生非预期的并发响应。

## F

### Focus Active State；焦点激活态

UI实例进入的一种方向键走焦可用状态，进入后用户可通过键盘方向键在可获焦组件间移动焦点，退出后方向键走焦不生效。它描述整个界面是否处于可用按键导航的整体模式；区别于单个组件是否获焦的局部状态。

### Focus Axis Event；焦点轴事件

与游戏手柄交互时，通过手柄十字按键或摇杆上报、经当前获焦组件分发回调给应用的轴事件。区别于基于指针位置的普通轴事件（鼠标滚轮、触控板），其输入源为游戏手柄且依赖组件焦点分发。

### Focus State；焦点态

组件处于获焦状态时展示的视觉反馈样式，通常表现为包裹组件边缘的焦点框（可配置边距、颜色、宽度），用于向键盘或遥控器用户明确标识当前输入指向的组件。焦点态样式仅在焦点激活态开启时生效。

### Frame Animation；帧动画

由多帧图像按时间顺序播放形成的动画形式，用于展示连续动态效果，适用于需要呈现动画序列的场景。

## G

### Gesture Resolution；手势裁决

在手势识别器即将识别成功时，由开发者回调根据当前事件信息与组件状态返回判定结果（如拒绝、继续），从而动态决定该识别器是否最终生效的机制。用于在运行时干预手势竞争结局（如嵌套滚动到达边界时屏蔽内层）。

## H

### Hit Test；命中测试

ArkUI框架在分发触屏、鼠标等交互事件前，将按压点与组件响应热区求交以收集需响应事件组件集合的过程，其结果直接决定后续事件分发的目标与顺序。是点击、触摸、拖拽、鼠标、轴、悬浮、手势等事件分发的前置命中判定环节；亦称触摸测试。

## I

### Imperative Node；命令式节点

相对于ArkUI的声明式语法，同时提供了一套命令式UI构建能力，开发者可使用ArkTS语言通过new FrameNode()等构造函数直接创建的节点，或使用C接口在Native侧构建UI。这种方式更加灵活，但也要求开发者更多参与UI管理以避免渲染错乱、内存泄漏等问题。在跨语言属性设置场景中，ArkTS侧创建的节点可由Native侧设置属性，C语言创建的节点可由ArkTS侧设置属性。

## L

### Long Tail State；长尾状态

弹簧动画主体运动结束后，逐渐衰减的微弱抖动阶段。此时动画看起来快停了但还未完全静止，是弹簧回弹特性的必然过程。

## M

### Mutual Exclusion Recognition；互斥识别

组合手势的一种识别模式，组内各手势同时识别，但只要其中任一手势率先识别成功，识别即结束、其余手势均判定失败。用于在一组相互冲突的手势中只让一个生效（如单击与双击二选一）。

## N

### Named Route；命名路由

通过名称标识而非URL路径进行页面跳转的路由方式，主要用于跨包（HAR或HSP共享包）重定向到包内页面。

## O

### off-screen rendering；离屏渲染

组件及其子组件先在屏幕外完成渲染，再与父控件一起绘制的处理方式；适用于需要把当前组件和子组件作为整体与父控件融合绘制的场景。

### Offscreen Snapshot；离屏截图

在应用后台对传入的自定义构建函数进行临时构建与渲染并输出其截图的能力，被截图的组件无需预先加载到当前组件树，适用于截取尚未显示在界面上的内容。区别于普通组件在线截图，由于需等待构建和渲染完成，其返回通常存在延迟。

### Onscreen Snapshot；在线截图

对已经在界面上渲染的组件进行截图的能力，被截图的组件需要已经渲染绘制到当前组件树，适用于截取已经显示在界面上的内容（可以不在当前屏幕可视范围内，但已经挂载在树上）。

## P

### Page Stack；页面栈

Router模块管理的页面跳转栈结构，记录页面进出顺序，最多容纳32个页面。

### passive observation；无感监听

在不侵入组件业务逻辑的情况下，通过Observer或UIObserver订阅UI组件行为变化、Router页面状态变化、Navigation页面切换、滚动、屏幕像素密度、绘制或布局等变化的机制；适用于需要在当前进程内获取相关变化信息并通过回调处理的场景。

### Polymorphic Style；多态样式

组件在不同交互、视觉状态下分别应用不同样式的机制，通过为正常态、按压态、禁用态、获焦态、点击态、选中态、悬浮态等分别配置样式集合，使同一组件呈现差异化外观。用于以声明式方式集中管理组件的状态化视觉表现。

### Pre-loading Area；预加载区域

懒加载模式下紧邻容器组件显示范围的区域，该区域内的子组件会在系统空闲时提前创建并布局以提升列表滑动性能。

## R

### Response Region；触摸热区

组件上能够响应触摸交互的有效区域，是命中测试中按压点与组件求交的判定依据，可与组件可见边界不一致（可扩展到可视边界之外或收缩）。用于扩大、缩小组件实际可点击范围、或将多个不连续区域绑定到同一组件，以满足触控目标尺寸要求；称响应区域。

### Route Map；路由表

Navigation中定义页面路由名称、源文件路径及自定义字段的配置映射，用于路由跳转时查找目标页面。

### Route Stack；路由栈

Navigation组件中由NavPathStack管理的NavDestination页面进出顺序栈结构，与Router管理的页面栈属不同导航模型。

## S

### Sequential Recognition；顺序识别

组合手势的一种识别模式，组内手势按注册顺序依次识别，前一手势必须识别成功才推进到下一个，若任一手势失败其后所有手势均无法完成。用于表达“先A后B”的串行交互链路（如先长按再拖动）。

## T

### Theme Skinning；主题换肤

应用内视觉主题切换机制，可定制应用的全局配色风格和局部配色风格。

## U

### UI context；UI上下文

UI实例的抽象运行环境，UI操作在该上下文中执行并最终反映到对应UI实例；在替换全局函数或处理非UI、异步场景时，用于显式定位接口应作用的目标UI实例。

### UI Parallelization；UI并行化

通过多线程创建UI组件、设置属性并将挂载任务提交到UI线程的页面构建方式，用于复杂业务场景下降低UI线程负载、减少页面创建耗时，并提升应用启动、页面更新和交互流畅度。

### UI thread；UI线程

运行ArkUI界面处理UI事件的线程，通常情况下是应用主线程，用于执行需绑定UI上下文的ArkUI接口；包括UI上下文环境初始化、销毁通知和帧回调注册等UI处理相关的接口必须在该线程调用，否则程序会主动中止。

## Z

### 组件坐标系

以组件左上角为坐标原点的坐标系，其中向右为x正轴，向下为y正轴。如果为三维坐标系，则由屏幕向外为z正轴。
