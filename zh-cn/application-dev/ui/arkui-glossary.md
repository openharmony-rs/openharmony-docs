## A

### Accessibility Virtual Node；无障碍虚拟子节点

当开发者使用自绘制组件进行自定义绘制时，这些组件内部的内容（如绘制的图形、文字）通常无法被无障碍服务直接感知。无障碍虚拟子节点是为自绘制组件补充的不可见无障碍节点，用于让辅助应用识别自绘区域内的可感知内容。

### Affine Property；仿射属性

组件的变换类属性（缩放、旋转、平移），基于组件坐标系和锚点进行仿射变换。组件平移后仿射属性的坐标系仍保持固定，与布局类属性（宽、高等）直接改变内容位置的方式不同。变更仿射属性不会触发测量布局，性能开销低于布局属性变更。

### Animatable Property；可动画属性

当属性值在可动画的闭包函数内发生变化时产生平滑的动画过渡而非直接跳变。区别于不可动画属性变化时瞬间生效无过渡。

## D

### dirty nodes；脏节点

状态变量变化后被标记为需要更新的关联节点，会在下一帧中更新并从脏节点列表清空；用于描述当前仍等待刷新的组件或节点状态。

### dirty state marking；标脏

组件状态或属性变化后，框架将对应组件或节点标记为需要更新的过程，用于在后续UI渲染中刷新标脏节点，并触发相关测量、布局和渲染；自定义属性或自定义绘制属性变化后，开发者需要主动通知框架重新测算或渲染。

## E

### Entity Node；实体节点

由后端创建的Native节点，是前端节点对象在底层的实际载体。前端节点通过引用关系与实体节点绑定，当调用dispose解除绑定后，前端节点不再对应任何实体节点，再次调用接口可能导致crash或返回默认值。

## F

### Frame Animation；帧动画

由多帧图像按时间顺序播放形成的动画形式，用于展示连续动态效果，适用于需要呈现动画序列的场景。

## I

### Imperative Node；命令式节点

相对于ArkUI的声明式语法，同时提供了一套命令式UI构建能力，开发者可使用ArkTS语言通过new FrameNode()等构造函数直接创建的节点，或使用C接口在Native侧构建UI。这种方式更加灵活，但也要求开发者更多参与UI管理以避免渲染错乱、内存泄漏等问题。在跨语言属性设置场景中，ArkTS侧创建的节点可由Native侧设置属性，C语言创建的节点可由ArkTS侧设置属性。

## L

### Long Tail State；长尾状态

弹簧动画主体运动结束后，逐渐衰减的微弱抖动阶段。此时动画看起来快停了但还未完全静止，是弹簧回弹特性的必然过程。

## N

### Named Route；命名路由

通过名称标识而非URL路径进行页面跳转的路由方式，主要用于跨包（HAR或HSP共享包）重定向到包内页面。

## O

### off-screen rendering；离屏渲染

组件及其子组件先在屏幕外完成渲染，再与父控件一起绘制的处理方式；适用于需要把当前组件和子组件作为整体与父控件融合绘制的场景。

## P

### Page Stack；页面栈

Router模块管理的页面跳转栈结构，记录页面进出顺序，最多容纳32个页面。

### passive observation；无感监听

在不侵入组件业务逻辑的情况下，通过Observer或UIObserver订阅UI组件行为变化、Router页面状态变化、Navigation页面切换、滚动、屏幕像素密度、绘制或布局等变化的机制；适用于需要在当前进程内获取相关变化信息并通过回调处理的场景。

### Pre-loading Area；预加载区域

懒加载模式下紧邻容器组件显示范围的区域，该区域内的子组件会在系统空闲时提前创建并布局以提升列表滑动性能。

## R

### Route Map；路由表

Navigation中定义页面路由名称、源文件路径及自定义字段的配置映射，用于路由跳转时查找目标页面。

### Route Stack；路由栈

Navigation组件中由NavPathStack管理的NavDestination页面进出顺序栈结构，与Router管理的页面栈属不同导航模型。

## T

### Theme Skinning；主题换肤

应用内视觉主题切换机制，可定制应用的全局配色风格和局部配色风格。

## U

### UI context；UI上下文

UI实例的抽象运行环境，UI操作在该上下文中执行并最终反映到对应UI实例；在替换全局函数或处理非UI、异步场景时，用于显式定位接口应作用的目标UI实例。

### UI Parallelization；UI并行化

通过多线程创建UI组件、设置属性并将挂载任务提交到UI线程的页面构建方式，用于复杂业务场景下降低UI线程负载、减少页面创建耗时，并提升应用启动、页面更新和交互流畅度。

### UI thread；UI线程

运行ArkUI界面处理UI事件的线程，通常情况下是应用主线程，用于执行需绑定UI上下文的ArkUI接口；包括UI上下文环境初始化、销毁通知和帧回调注册等UI处理相关的接口必须在该线程调用，否则程序会主动中止。

## 组件坐标系

以组件左上角为坐标原点的坐标系，其中向右为x正轴，向下为y正轴。如果为三维坐标系，则由屏幕向外为z正轴。
