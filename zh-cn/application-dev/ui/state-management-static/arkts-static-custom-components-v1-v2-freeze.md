# 自定义组件冻结

组件冻结功能是一种性能优化机制，它会冻结非激活状态下的组件的刷新能力。当组件处于非激活状态时，即使其绑定的状态变量发生变化，也不会触发该组件的UI重新渲染，从而降低复杂UI场景下的刷新负载。

> **说明：**
>
> 从API version 23开始，支持自定义组件冻结功能。
>

## 概述

在ArkTS-Sta上下文中，自定义组件默认打开组件冻结机制。系统将仅对处于激活状态的自定义组件进行更新，这使得UI框架可以尽量缩小更新范围，仅限于用户可见范围内（激活状态）的自定义组件，从而提高复杂UI场景下的刷新效率。当处于非激活状态的自定义组件重新变为激活状态时，状态管理框架会对其执行必要的刷新操作，确保UI的正确展示。

组件冻结会同时冻结状态变量的变化观察，即V1自定义组件中@Watch、V2自定义组件中@Monitor不会立刻触发，而是当组件重新变为激活状态时才会触发。

简而言之，组件冻结旨在优化复杂界面下的UI刷新性能。在存在多个不可见自定义组件的情况下，如多页面栈、长列表或宫格，通过组件冻结可以实现按需刷新，即仅刷新当前可见的自定义组件，而将不可见自定义组件的刷新延迟至它们变为可见时。

状态管理V1与V2皆支持自定义组件冻结机制，但支持场景略有不同，参考以下冻结场景文档。
