## 模块简介

AVSource 模块提供用于构造媒体资源对象功能的 C API。开发者通过本模块可以根据 URI、文件描述符或自定义数据源创建媒体源实例，获取媒体文件的基础信息和轨道信息，为后续解封装操作提供数据源。

本模块提供的主要能力包括：

- **媒体源创建与销毁**：根据 URI、文件描述符或用户自定义数据源创建媒体源实例，销毁媒体源实例释放资源。
- **文件基础信息获取**：获取媒体资源文件的基础信息，包括文件时长、轨道数量、文件类型等。
- **轨道信息获取**：获取指定轨道的基础信息，包括轨道类型、编码格式、分辨率、帧率、采样率、声道数等。
- **自定义元数据获取**：获取媒体文件中的自定义元数据信息。

适用场景包括：媒体播放器开发、视频编辑、音视频数据分析、媒体文件解析等需要访问媒体文件元数据和轨道信息的场景。

本模块通常作为媒体处理流程的入口，与 AVDemuxer 模块配合使用：先通过 AVSource 创建媒体源并获取轨道信息，再通过 AVDemuxer 进行解封装读取压缩数据。

本模块包含头文件 `native_avsource.h`，提供结构体 `OH_AVSource`，以及创建、销毁、信息获取等函数。

## 关键术语

### 媒体资源术语

#### Media Source；媒体资源
音视频数据的抽象表示，封装了媒体文件的位置、格式和访问方式等信息，用于后续的解封装和播放处理。

#### Data Source；数据源
媒体数据的来源方式，包括文件描述符、统一资源标识符（URI）、用户自定义回调等，决定了媒体数据的读取方式。

#### URI (Uniform Resource Identifier)；统一资源标识符
用于标识网络资源位置的字符串，在媒体场景中指向远程音视频文件的访问地址，支持HTTP、HTTPS等协议。

#### FD (File Descriptor)；文件描述符
操作系统分配给打开文件的整型标识符，用于通过文件系统访问本地媒体文件，支持指定读取位置和范围。

#### Progressive Streaming；渐进式流媒体
一种流媒体传输方式，按顺序从网络下载媒体数据，下载完成后可播放，不支持动态码率切换，适用于网络稳定的场景。

#### HLS (HTTP Live Streaming)；HTTP直播流
Apple公司开发的流媒体传输协议，将媒体流分割为小段并通过HTTP传输，支持自适应码率和实时直播，广泛用于苹果生态系统。

#### DASH (Dynamic Adaptive Streaming over HTTP)；动态自适应流
一种基于HTTP的自适应流媒体传输协议，根据网络条件动态选择合适的码率分段，提供流畅的播放体验，是国际标准方案。

### 轨道与格式术语

#### Track Index；轨道索引
媒体文件中特定轨道的编号标识，从0开始顺序递增，用于在多轨道媒体文件中定位和访问特定轨道。

#### Source Format；源格式
媒体资源文件级别的格式信息，包括文件封装格式、总时长、轨道数量、全局元数据等，描述整个媒体文件的基本属性。

#### Track Format；轨道格式
单个媒体轨道的格式信息，包括编解码类型、分辨率、采样率、声道数等，描述特定轨道的技术参数。

#### Custom Metadata；自定义元数据
媒体文件中非标准的扩展信息，由内容制作者添加的附加描述数据，如地理位置、拍摄参数、版权声明等。

### 解封装相关术语

#### Demuxer；解封装器
从封装格式中提取音视频数据的组件，将容器文件分离为独立的音频轨、视频轨和字幕轨，供解码器处理。

#### Container Format；封装格式
音视频数据的容器格式，将音频、视频、字幕等数据打包存储为单一文件，如MP4、MKV、FLV等格式。

#### Spliter；分离器
将封装的媒体流分离为各轨道原始数据的处理单元，是解封装器的核心组件，支持多种封装格式的解析。
