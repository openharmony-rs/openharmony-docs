## 概述

本文档介绍剪贴板模块相关的错误码，帮助开发者定位和解决剪贴板操作过程中遇到的问题。文档涵盖了索引操作、容量限制、操作冲突、权限设置、系统异常等场景的错误码说明。

> **说明：**
>
> 以下仅介绍本模块特有错误码，通用错误码请参考[通用错误码说明文档](../errorcode-universal.md)。

## 12900001 索引超过范围

**错误信息**

The index is out of range.

**错误描述**

当调用getRecord等涉及索引的接口时，若索引超过范围（应介于0至recordCount-1之间），系统会报此错误码。

**可能原因**

接口的参数索引值超出当前PasteData中的记录数，比如，getRecord传入index值过大。

**处理步骤**

1. 检查参数索引值是否在正确范围（应介于0至recordCount-1之间），可通过调用getRecordCount()方法获取当前记录数。
2. 使用恰当的索引值重新进行开发。

## 12900002 Record数量超过最大限制

**错误信息**

The number of records exceeds the upper limit.

**错误描述**

当添加Record时，若当前PasteData记录数已达到最大值，系统会报此错误码。

**可能原因**

由于未进行相关Record的删除或数目检查，在记录数已达最大值时，若继续添加Record，系统会报此错误码。

**处理步骤**

1. 检查判断当前PasteData记录数是否已达最大值。
2. 若当前PasteData记录数已达最大值，删除相关Record后，再重新添加Record进行开发。

## 12900003 另外一个复制或粘贴正在进行

与错误码[27787277](#27787277-另外一个复制或粘贴正在进行)含义相同。

**错误信息**

Another copy or paste operation is in progress.

**错误描述**

上次的复制/粘贴动作还未结束时，再次调用相关接口，系统会报此错误码。

**可能原因**

当复制/粘贴的数据内容较大，需要时间较长时，在此期间再次执行复制/粘贴则会触发此错误码。

**处理步骤**

1. 再次进行复制/粘贴时，首先通过回调或Promise判断上次复制/粘贴的状态是否已完成。
2. 确保在上次复制/粘贴操作完成后，再进行新的复制/粘贴操作。

## 12900004 禁止复制

与错误码[27787278](#27787278-禁止复制)含义相同。

**错误信息**

Replication is prohibited.

**错误描述**

对不支持复制的数据内容进行复制操作时，系统会报此错误码。

**可能原因**

数据内容不支持复制，比如只读类型的数据内容。

**处理步骤**

1. 在对相关数据内容进行复制时，首先判断其数据类型是否支持复制。
2. 若数据内容不支持复制（如只读类型数据），不对此类数据内容进行复制操作。如DLP（Data Loss Prevention，数据防泄漏）只读数据，参见[数据防泄漏服务简介](../../security/DataProtectionKit/dlp-overview.md)。

## 12900005 请求超时

**错误信息**

Excessive processing time for internal data.

**错误描述**

当内部处理数据耗时过长时，系统会报此错误码。

**可能原因**

内部处理数据耗时过长，比如处理数据过大。

**处理步骤**

建议处理数据过大时，使用异步接口替代同步接口，以避免超时错误。

## 12900006 设置已存在

**错误信息**

Settings already exist.

**错误描述**

当应用全局的可粘贴的范围已存在时，再次进行设置会导致系统报此错误码。

**可能原因**

应用全局的可粘贴的范围已存在。

**处理步骤**

1. 调用相关方法删除已有的可粘贴范围设置。
2. 确认删除成功后，再调用设置方法进行新的设置。

## 12900007 文件拷贝失败

**错误信息**

Invalid destUri or file system error.

**错误描述**

当文件拷贝失败时，系统会报此错误码。

**可能原因**

1. 目标路径（destUri）无效或不存在。
2. 文件系统出现错误，如存储空间不足、权限受限等。

**处理步骤**

1. 在对相关数据内容进行拷贝时，首先判断目标路径是否有效。
2. 若目标路径不支持拷贝时，请确认正确的路径。

## 12900008 启动进度条hap失败

**错误信息**

Failed to start progress.

**错误描述**

当进度条指示选项为系统默认进度时，如果创建进度显示失败，系统会报此错误码。当未自定义进度显示时，系统会自动使用默认进度条。自定义进度条显示请参考[ProgressIndicator](./js-apis-pasteboard.md#progressindicator15)。

**可能原因**

系统线程异常。

**处理步骤**

确认是否粘贴成功。

1. 若粘贴成功可忽略此错误码。
2. 若粘贴失败需要进一步排查原因。
3. 如问题持续，请联系技术人员获取支持。

## 12900009 进度上报异常

**错误信息**

Progress exits abnormally.

**错误描述**

采用系统默认进度显示（当未自定义进度显示时，系统会自动使用默认进度条），进度上报异常时，系统会报此错误码。

**可能原因**

进度显示异常，可能原因包括：系统资源不足、UI线程阻塞、进度条组件异常等。

**处理步骤**

1. 检查系统资源状态，如内存、CPU占用情况，确认是否存在资源不足。
2. 检查UI线程是否存在阻塞，避免在UI线程执行耗时操作。
3. 若粘贴成功可忽略此错误码，不影响正常使用。
4. 若粘贴失败需要进一步确认，请联系技术人员获取支持。

## 12900010 获取粘贴数据失败

**错误信息**

System error occurred during paste execution.

**错误描述**

当获取粘贴数据失败时，系统会报此错误码。

**可能原因**

系统内部错误导致获取粘贴数据失败，可能包括：系统资源不足、权限不足、剪贴板服务异常、数据格式不兼容或内存不足等情况。

**处理步骤**

1. 检查应用权限配置，确保已获取必要的剪贴板访问权限。
2. 检查系统资源状态，确认是否存在资源不足的情况。
3. 若问题持续存在，请联系技术人员获取支持并提供详细的错误日志。

**错误描述**

**可能原因**

**处理步骤**

1. 再次进行复制/粘贴时，首先通过回调或Promise判断上次复制/粘贴的状态是否已完成。
2. 确保在上次复制/粘贴操作完成后，再进行新的复制/粘贴操作。
