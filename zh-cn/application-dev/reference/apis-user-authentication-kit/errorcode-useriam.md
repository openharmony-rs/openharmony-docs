本文档介绍用户认证服务特有的错误码。帮助开发者定位和处理认证相关的异常情况。

> **说明：**
>
> 以下仅介绍本模块特有错误码，通用错误码请参考[通用错误码说明文档](../errorcode-universal.md)。

## 12500001 认证不通过

**错误信息**

Authentication failed.

**错误描述**

用户身份认证不通过，表示用户提供的认证凭据与设备中已注册的凭据比对不匹配。常见的认证结果，通常表示用户输入错误或使用了未注册的凭据。

**可能原因**

1. 用户输入的密码、人脸或指纹与已注册的凭据不匹配。
2. 用户使用了未注册的认证凭据尝试认证。
3. 生物特征采集质量不佳，导致比对失败（如人脸光线不足、指纹按压不完整等）。

**处理步骤**

1. 建议提示用户重新尝试认证。
2. 对于生物认证失败，可提示用户调整操作方式（如改善光线环境、正确按压指纹传感器）。
3. 如果多次失败，建议用户检查是否已注册相应凭据，或切换到其他认证方式。
4. 记录失败次数，当达到阈值时提示用户可能触发锁定机制。

## 12500002 身份认证系统通用错误码

**错误信息**

General operation error.

**错误描述**

身份认证系统发生无法恢复的内部错误。通用错误码，表示系统内部工作异常，开发者无法通过调整参数或操作方式来解决。

**可能原因**

1. NAPI层解析参数出错，参数传递过程中发生异常。
2. 用户认证服务的进程没有启动或意外终止。
3. IPC通信的proxy客户端写数据出错，数据传输异常。
4. IPC通信的stub服务端解析数据出错，数据接收异常。
5. 获取driver服务出错，底层驱动服务不可用。
6. 系统资源不足，内存分配失败。

**处理步骤**

1. 系统服务内部工作异常，建议重新调用接口。
2. 如果多次重试仍然失败，建议重启设备尝试。

## 12500003 认证被取消

**错误信息**

Authentication canceled.

**错误描述**

认证操作被取消，表示用户主动或被动终止了认证过程。

**可能原因**

1. 用户手动点击取消按钮或返回键，主动取消认证。
2. 应用调用cancel接口主动取消认证。
3. 后续发起的认证请求优先完成，致使前置认证请求被强制取消。

**处理步骤**

1. 根据用户意图决定是否重新发起认证。
2. 如果是系统抢占取消，建议稍后重新发起认证请求。
3. 避免同时发起多个认证请求，防止相互抢占。

## 12500004 认证操作超时

**错误信息**

Authentication timeout.

**错误描述**

认证操作超过设定的时限。用户在规定时间内未完成认证交互，系统自动终止认证过程。

**可能原因**

1. 用户在认证界面停留过久，未及时输入密码或进行生物认证操作。
2. 人脸认证时用户未正视摄像头，导致长时间无法采集有效人脸图像。

**处理步骤**

1. 建议用户重新发起认证，并注意操作时限。
2. 提示用户在认证时及时响应，如人脸认证时正视摄像头。
3. 对于密码认证，提醒用户在超时前完成密码输入。

## 12500005 认证类型不支持

**错误信息**

The authentication type is not supported.

**错误描述**

表示当前设备或系统不支持请求的认证方式。

**可能原因**

1. 传入的认证类型参数无效，例如调用[getAvailableStatus](js-apis-useriam-userauth.md#userauthgetavailablestatus9)接口时传入的authType参数不在支持的范围内（PIN、FACE、FINGERPRINT等）。
2. 传入的认证类型参数在本设备上不支持，例如在没有部署指纹传感器的设备上发起指纹认证。

**处理步骤**

1. 请检查传入的认证类型参数是否正确，确保参数值在支持的范围内。
2. 发起认证之前，调用[getAvailableStatus](js-apis-useriam-userauth.md#userauthgetavailablestatus9)预先检查设备是否支持指定的认证类型，如果设备不支持某种认证类型，建议切换到其他支持的认证类型。

## 12500006 认证信任等级不支持

**错误信息**

The authentication trust level is not supported.

**错误描述**

表示当前认证类型无法达到请求的安全等级。

**可能原因**

1. 调用[getAvailableStatus](js-apis-useriam-userauth.md#userauthgetavailablestatus9)或[getUserAuthInstance](js-apis-useriam-userauth.md#userauthgetuserauthinstance10)接口时，传入的authTrustLevel参数不在有效范围内（应为ATL1(10000)、ATL2(20000)、ATL3(30000)或ATL4(40000)）。
2. 当前设备的认证能力无法达到指定的认证信任等级，例如在2D人脸认证设备上发起ATL4等级的人脸认证。
3. 用户注册的凭据安全等级低于请求的认证信任等级，如4位PIN密码只能达到ATL3，无法满足ATL4要求。

**处理步骤**

1. 检查传入的authTrustLevel参数是否在有效范围内（ATL1-ATL4）。
2. 如果参数在有效范围内，则可能是当前设备不支持该认证信任等级，建议应用降低认证等级。
3. 调用getAvailableStatus接口预先检查设备是否支持指定的认证信任等级。
4. 如果业务需要高安全等级，可考虑使用支持更高等级的认证类型（如使用PIN代替人脸认证）。
5. 参见[认证可信等级划分原则](../../security/UserAuthenticationKit/user-authentication-overview.md#生物认证可信等级划分原则)了解各认证类型支持的等级范围。

## 12500007 认证服务繁忙

**错误信息**

Authentication service is busy.

**错误描述**

认证服务正忙于处理其他请求，无法响应当前的认证请求。表示系统认证服务暂时不可用。

**可能原因**

1. 认证服务正在处理其他应用的认证请求，资源被占用。
2. 系统正在进行其他认证相关操作（如设备解锁），无法响应新的认证请求。
3. 认证服务启动过程中暂时无法响应请求。

**处理步骤**

1. 如果是服务繁忙，建议稍后重新发起认证请求。
2. 避免在短时间内频繁发起认证请求，可增加请求间隔时间。

## 12500008 参数校验失败

**错误信息**

The parameter is out of range.

**错误描述**

参数取值超出合法范围，表示传入的参数值不符合接口定义的有效范围。

**可能原因**

1. 参数值超出定义的有效范围，如reuseDuration超过MAX_ALLOWABLE_REUSE_DURATION（300000毫秒）。
2. 数值类型参数为负数或零，但接口要求为正整数。
3. 字符串参数长度超过限制，如title超过500字符、navigationButtonText超过60字符。
4. 数组参数为空但接口要求非空，如authType列表为空。
5. Uint8Array参数长度超过限制，如challenge超过32字节、authToken超过1024字节。

**处理步骤**

1. 参考接口文档中的参数说明，确认各参数的有效取值范围，修正参数后重新发起请求。
2. 对于数值参数，确保在有效范围内且符合类型要求。
3. 对于字符串参数，确保长度不超过限制且内容符合要求。
4. 对于数组参数，确保不为空且元素有效。
5. 对于二进制数据参数，确保长度在有效范围内。

## 12500009 认证被锁定

**错误信息**

Authentication is locked out.

**错误描述**

指定认证方式已被锁定，表示因连续多次认证失败触发防暴保护机制，认证功能暂时不可用。

**可能原因**

用户频繁尝试错误的认证凭据，系统启动保护机制防止暴力破解。

**处理步骤**

1. 提示用户认证已锁定，需等待锁定解除后才能继续认证。
2. 可通过[getAuthLockState](js-apis-useriam-userauth.md#userauthgetauthlockstate22)接口查询当前锁定状态，包括剩余锁定时间和剩余尝试次数。
3. 如果是临时锁定，提示用户等待锁定时间到期后重新尝试。
4. 如果是永久锁定，提示用户使用密码解锁后才能继续使用生物认证。
5. 建议用户在锁定解除后使用正确的凭据进行认证，避免再次触发锁定。

## 12500010 该类型的凭据没有录入

**错误信息**

The type of credential has not been enrolled.

**错误描述**

用户未录入指定类型的认证凭据，表示设备中没有注册请求的认证方式对应的凭据数据。

**可能原因**

1. 调用userAuth模块的[getAvailableStatus](js-apis-useriam-userauth.md#userauthgetavailablestatus9)接口检查认证能力时，传入的authType参数为FACE或FINGERPRINT，但用户未在设备上录入人脸或指纹凭据。
2. 调用start接口发起认证时，请求的认证类型对应的凭据未录入，如发起指纹认证但用户未录入指纹。
3. 用户曾录入的凭据已被删除，如用户在设置中删除了已注册的人脸或指纹。
4. 新用户账号未录入任何认证凭据。

**处理步骤**

1. 检查用户是否已录入该类型的认证凭据。
2. 如果用户未录入，引导用户前往系统设置录入相应凭据（人脸、指纹或PIN）。
3. 可通过[getEnrolledState](js-apis-useriam-userauth.md#userauthgetenrolledstate12)接口查询凭据注册状态。
4. 如果用户拒绝录入，建议切换到其他已录入的认证类型。
5. 提示用户录入凭据后才能使用相应的认证功能。

## 12500011 提示/通知切换自定义认证

**错误信息**

Switched to the custom authentication process.

**错误描述**

用户取消系统认证方式，选择使用应用自定义认证。

**可能原因**

用户在身份认证控件界面点击了导航按钮（由navigationButtonText参数指定），当前的系统认证操作被用户取消，选择使用应用自定义认证方式。

**处理步骤**

应用收到此错误码后，需拉起自定义认证界面供用户完成认证。

## 12500013 密码过期

**错误信息**

Operation failed because of PIN expired.

**错误描述**

密码过期，表示密码因安全策略要求已过期。

**可能原因**

1. 系统锁屏口令已过期，通常由企业安全策略或系统设置导致（如要求定期更新密码）。
2. 用户发起密码认证、指纹认证或人脸认证时，系统检测到锁屏密码已过期。
3. 密码过期后，生物认证（指纹、人脸）也无法使用，因为它们依赖密码作为备用解锁方式。

**处理步骤**

1. 提示用户密码已过期，需更新密码后才能继续使用认证功能。
2. 引导用户前往系统设置更新密码。
3. 用户更新密码后，可重新发起认证请求。
4. 如果是企业环境，可能需要联系管理员了解密码更新要求。
5. 提醒用户定期更新密码以避免过期影响使用。

## 12500015 AuthToken完整性校验失败

**错误信息**

Operation failed because of authToken integrity check failed.

**错误描述**

AuthToken完整性校验失败，表示验证的认证令牌无效或已被篡改。令牌的数字签名或数据完整性校验失败，无法确认令牌的真实性。

**可能原因**

1. 验证的AuthToken不是由系统签发的有效令牌，可能是伪造的令牌。
2. AuthToken在传输过程中被篡改，数据完整性被破坏。
3. AuthToken的数字签名校验失败，签名不匹配或签名算法错误。
4. AuthToken格式错误，不符合系统定义的令牌格式。
5. AuthToken已失效，如用户凭据变更导致令牌失效。

**处理步骤**

1. 重新发起认证请求，获取系统签发的有效AuthToken。
2. 确保AuthToken在传输过程中未被修改，使用安全的数据传输方式。
3. 如果凭据已变更，需重新认证获取新的AuthToken。
4. 参考[verifyAuthToken](js-apis-useriam-useraccessctrl-sys.md#useraccessctrlverifyauthtoken)接口了解AuthToken校验流程。

## 12500016 AuthToken过期

**错误信息**

Operation failed because of authToken has expired.

**错误描述**

AuthToken已过期，表示认证令牌的签发时间至验证时的时间间隔超过了允许的最大有效时长。

**可能原因**

1. AuthToken的签发时间至发起验证时的时间间隔超过了传入的最大有效时长（allowableDuration）参数。
2. AuthToken使用时间过长，超过了系统设定的最大有效期（24小时）。
3. 应用在AuthToken即将过期时才发起验证，导致验证时已超时。
4. 复用AuthToken的最大有效时长参数过短，导致令牌提前过期。原因是AuthToken签发时间为原认证时间，而非当前调用时间。

**处理步骤**

1. 重新发起认证请求，获取新的有效AuthToken。
2. 在AuthToken有效期内尽快使用，避免延迟导致过期。
3. 根据业务场景合理设置allowableDuration参数，确保满足业务需求。

## 12500017 复用身份认证结果失败

**错误信息**

Failed to reuse authentication result.

**错误描述**

复用身份认证结果失败，表示系统无法找到满足复用条件的认证结果。

**可能原因**

1. 认证类型与指定的复用模式不匹配，如请求复用人脸认证结果但之前认证使用的是指纹。
2. 认证结果超过了可复用时长（可复用时长最长为5分钟），之前的认证结果已失效。
3. 设备解锁后或距离上一次认证超过了可复用时长。
4. 凭据在复用时长内发生了变更，如用户删除或添加了指纹、人脸凭据，导致之前的结果无法复用。
5. 之前的认证信任等级低于当前请求的等级，无法满足复用条件。
6. 没有设备解锁或身份认证的历史记录可供复用。

**处理步骤**

1. 发起新的认证请求，让用户手动完成认证以获取有效的AuthToken。
2. 检查reuseUnlockResult参数配置是否正确，包括reuseMode和reuseDuration。
3. 如果需要复用认证结果，确保在有效时长内发起请求（最长5分钟）。
4. 参考[认证结果复用](js-apis-useriam-userauth.md#reuseunlockresult12)了解复用机制和条件，选择合适的ReuseMode，设置合理的有效时长。

## 12700001 人脸服务不可用

**错误信息**

The service is unavailable.

**错误描述**

人脸服务不可用，表示人脸录入或认证所需的服务未正常运行。

**可能原因**

1. 调用faceAuth模块的[setSurfaceId](js-apis-useriam-faceauth-sys.md#setsurfaceid)接口时，人脸认证服务未启动。
2. IPC通信的proxy客户端写数据出错，数据传输异常。
3. IPC通信的stub服务端解析数据出错，数据接收异常。
4. 人脸driver层调用出错，底层驱动服务异常。
5. 摄像头硬件服务不可用，无法进行人脸图像采集。
6. 人脸服务进程崩溃或被意外终止。

**处理步骤**

1. 系统服务内部工作异常，建议稍后重新调用接口。
2. 如果多次重试仍然失败，建议重启设备尝试。
3. 检查摄像头硬件是否正常工作，其他应用是否能正常使用摄像头。
4. 检查系统日志，确认是否有服务崩溃或驱动异常的记录。

## 32600001 系统服务工作异常

**错误信息**

The system service is not working properly. Please try again later.

**错误描述**

伴随设备认证系统服务不可用，表示系统内部工作异常，无法响应认证相关请求。

**可能原因**

1. 伴随设备认证服务的进程没有启动或意外终止。
2. IPC通信的proxy客户端写数据出错，数据传输异常。
3. IPC通信的stub服务端解析数据出错，数据接收异常。
4. 系统资源不足，服务无法正常运行。
5. 服务启动过程中暂时无法响应请求。

**处理步骤**

1. 系统服务工作异常，建议稍后重新调用接口。
2. 如果多次重试仍然失败，建议重启设备尝试。
3. 检查系统日志，确认是否有服务崩溃或资源不足的记录。
4. 向系统管理员或技术支持报告此问题。

## 32600002 模板未找到

**错误信息**

The template is not found.

**错误描述**

伴随设备认证模板未找到，表示指定的模板ID不存在或已被删除。

**可能原因**

1. 传入的模板ID错误，不存在对应的模板记录。
2. 模板已被删除，如用户删除了已注册的伴随设备。
3. 模板ID格式错误，不符合系统定义的ID格式。
4. 查询的用户下没有注册任何伴随设备模板。

**处理步骤**

1. 在更新业务范围或订阅认证状态前，先确认模板存在。
2. 如果模板已被删除，需重新添加伴随设备并注册模板。
3. 确保传入的模板ID与用户注册的模板匹配。
4. 确认模板ID是否正确，可通过[getTemplateStatus](js-apis-useriam-companiondeviceauth-sys.md#gettemplatestatus)查询当前用户的模板列表。

## 32600003 业务ID无效

**错误信息**

The business id is invalid.

**错误描述**

业务ID无效，表示传入的业务ID不在支持的范围内或不符合格式要求。

**可能原因**

1. 传入的业务ID错误，不在系统支持的业务范围内。
2. 业务ID值小于系统保留值范围（0-9999），与系统预定义的业务ID冲突。
3. 业务ID值大于VENDOR_BEGIN(10000)但未在应用中定义。
4. 传入的业务ID数组中包含无效值。
5. 业务ID格式错误，应为整数类型。

**处理步骤**

1. 确认业务ID是否正确，参考[BusinessId](js-apis-useriam-companiondeviceauth-sys.md#businessid)枚举了解系统支持的业务ID。
2. 使用系统预定义的业务ID（如DEFAULT）或厂商自定义ID（≥10000）。
3. 确保自定义业务ID在应用中有明确定义，避免与系统保留值冲突。
